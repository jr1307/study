javascript 와 jQuery 비교

접근방법

자바스크립트는 메서드를 이용해서 일일히 접근해야 하는 불편함이 있으며, IE하위버전에 지원하지 않는 기능들이 있어 크로스브라우징이 어렵다.

제이쿼리는 css작성방법과 동일하게 원하는 Element에 접근이 용이하며 크로스브라우징이 쉽다.

  • javascript selector : getElementsById(), getElementsByTagName(), getElementsByClassName()
  • jQuery selector : $('#id'), $('.class')

javascript querySelector()

- jquery처럼 괄호안에 css 선택자와 동일하게 넣어 작성할 수 있다. 예) document.querySelector('.class')

- 괄호안에 명시된 선택자가 여러개라도 그중 가장 앞에 있는 것만 선택된다.

- IE8부터 지원

javascript querySelectorAll()

- jquery처럼 괄호안에 css 선택자와 동일하게 넣어 작성할 수 있다. 예) document.querySelectorAll('.class')

- 괄호안에 명시된 선택자가 모두 선택된다. 다수의 Element가 선택되므로 Index번호를 통해 접근이 가능하다.

- IE9부터 지원

javascript .firstChild / .firstElementChild

- firstChild는 element와 text와 주석을 모두 인식하여 가장 앞에 있는것을 반환한다.

- firstElementChild는 첫번째 element를 반환한다.

- firstElementChild는 IE9부터 지원

javascript .lastChild / .lastElementChild

- lastChild element와 text와 주석을 모두 인식하여 가장 뒤에 있는것을 반환한다.

- lastElementChild 마지막 element를 반환한다.

- lastElementChild는 IE9부터 지원
